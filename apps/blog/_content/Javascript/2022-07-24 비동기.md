비동기 프로그래밍이 Javascript에서 굉장히 중요하다는 것을 느꼈다. 자바스크립트는 싱글 스레드로 동작하는 언어이기 때문에, 비동기 프로그래밍 기법이 없었다면 아마 통신에서 처리하는 과정들이 모두 쌓이다가 병목현상(태스크가 적체되는 현상)이 나타나게 됐을 것이다.

비동기로 처리하기 위해서는 Callback 함수, Promise(), async/await 다양한 방식이 존재하는데, 이번에는 이러한 방식들이 어떻게 동작하는지 한번 살펴보자.

---

## 실행 컨텍스트와 콜스택

함수를 호출 하면 함수 실행 컨텍스트가 실행됩니다. 이때 실행 컨텍스트는 실행 컨텍스트 스택(이게 콜스택)에 푸쉬되고 함수가 실행됩니다.

> 다른 말로하면, 콜스택에 함수 실행 컨텍스트가 푸쉬되는 것은 함수 실행의 시작을 의미합니다.

### 동기와 비동기

여기서 중요한 점이 있습니다. **자바스크립트 엔진은 단 하나의 실행 컨텍스트 스택을 가진다는 것입니다**. 이 말은 2 개 이상의 함수를 동시에 실행할 수 없다는 뜻 입니다. 이를 싱글 스레드 방식이라고 부릅니다.

이런 싱글 스레드 방식은 처리 시간이 걸리는 태스크를 실행하는 경우 블로킹 현상이 발생합니다. 이렇게 동기적으로 작업을 진행한다면, 앞선 태스크가 종료될 때까지 기다리게 됩니다.

![](https://velog.velcdn.com/images/willy4202/post/e88aad26-e342-4a7b-88d0-f47b137ec93f/image.png)

이러한 동기 처리 방식은 태스크를 순서대로 처리하기에, 실행 순서가 보장된다는 장점이 있습니다. 그러나 이후 태스크들은 블로킹된다는 단점을 알아보았습니다.

비동기 처리 방식은 블로킹이 발생하지 않지만, 실행 순서가 보장되지 않는 단점이 있습니다.

## 비동기 처리 방식

이렇게 비동기 처리를 수행하기 위해선 세가지 방법이 있습니다.

1. callback
2. promise
3. async/await

여기서 비동기 처리를 위한 콜백패턴은 콜백 헬을 만들 수 있어 가독성이 좋지 않고, 예외 에러 처리가 어렵고, 여러가지의 비동기 처리를 받기도 어렵습니다.

여기서는 setTimeout, setInterval, HTTP 요청, 이벤트 핸들러에 대해서 알아봅시다. 이 것들은 비동기 처리 방식으로 동작하는데, **이벤트 루프** 그리고 **태스크 큐**에 대해서 알아야합니다.

## 이벤트 루프와 태스크 큐

Javascript는 싱글 스레드 방식으로 하나의 태스크를 처리한다고 하지만, 일반 유저 입장에서 브라우저를 이용할 땐, 많은 태스크가 동시에 처리되는 것처럼 느껴집니다.

HTML요소가 움직이며, 데이터를 가져오면서 렌더링을 합니다. 이처럼 Javascript의 동시성을 지원하는 것이 **이벤트 루프**입니다.

이벤트 루프는 브라우저에 내장되어 있는 기능입니다. 대부분의 Javascript 엔진은 크게 두가지로 나뉩니다.

![](https://velog.velcdn.com/images/willy4202/post/eaa6d33b-70b4-4dc9-8a3e-db60ce068dca/image.png)

#### 콜스택, call stack

소스코드 평가 과정에서 생성된 실행 컨텍스트가 추가되고 제거되는 스택 자료구조인 실행 컨텍스트의 스택을 뜻합니다.

#### 힙,heap

힙은 객체가 저장되는 메모리 공간입니다. 콜 스택의 요소인 실행 컨텍스트는 힙에 저장된 객체를 참조합니다.

이처럼 콜 스택과 힙으로 구성되어 있는 Javascript 엔진은 단순히 태스크가 요청되면 콜스택을 통해 작업을 순차적으로 실행합니다.

비동기 처리에서 소스코드의 평가와 실행을 제외한 모든 처리는 Javascript 엔진을 구동하는 환경인 `브라우저`, `Node.js`가 담당합니다.

예를 들어, `setTimeout()`의 콜백 함수의 평가와 실행은 자바스크립트 엔진이 담당하지만 호출 스케줄링을 위한 타이머 설정과 콜백 함수의 등록은 `브라우저`, `Node.js`가 담당합니다. 이를 위해 브라우저 환경은 태스크 큐와 이벤트 루프를 제공합니다.

### 태스크 큐

`setTimeout()`,`setInterval`과 같은 비동기 함수의 콜백 함수 또는 이벤트 핸들러가 일시적으로 보관되는 영역입니다.

이때 태스크 큐는 FIFO 형태로 저장되며 먼저 들어온 것이 먼저 빠저나가는 방식입니다.

### 이벤트 루프

이벤트 루프는 콜 스택에 현재 실행 중인 실행 컨텍스트가 있는지, 그리고 태스크 큐에 대기 중인 함수가 있는지를 반복해서 확인합니다.

만약 콜 스택이 비어있고 태스크 큐에 대기 중인 함수가 있다면 이벤트 루프는 순차적으로 태스크 큐에 대기중인 함수를 콜 스택으로 이동시킵니다.

> 비동기 함수의 콜백함수는 태스크 큐에 푸시되어 대기하다가, 콜 스택이 비게 되면 (명시적으로 호출된 함수가 모두 종료하면) 콜 스택에 푸시되어 실행됩니다.

### 브라우저와 자바스크립트 엔진의 차이점

조금 헷갈릴 수 있을 것입니다. 앞서, Javascript는 싱글 스레드 방식으로 작동한다고 설명드렸습니다. 하지만 브라우저는 싱글 스레드로 작동하지 않습니다.

> 만약 모든 Javascript 코드가 엔진에서 싱글 스레드 방식으로 동작한다면, Javascript는 비동기로 동작할 수 없습니다. **Javascript 엔진은 싱글 스레드로 동작하지만, 브라우저는 멀티 스레드로 동작합니다!**
