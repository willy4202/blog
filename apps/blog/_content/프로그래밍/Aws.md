최근 채용 공고에서 개발자 우대 사항에 '서버를 배포해본 경험이 있는 사람'이 적혀있을 정도로 시스템 관리에 대한 경험을 중요하게 생각하는 것을 볼 수 있다.

개발자 입장에서 개발하고 있는 제품을 로컬에서뿐만 아니라, 다른 사람들도 접속할 수 있게 만들기 위해서 서버를 배포한다고 한다. 그러나 이 모든 과정을 진행하기 위해서는 서버실도 있어야하고, 이를 유지 보수할 인원도 필요하다.

이 과정을 획기적으로 줄이고, 누구나 서버를 배포할 수 있도록 도와주는 툴이 있다고 한다. 바로 AWS다. AWS는 어떤 것인지 자세히 알아보고, 위 기술을 사용하기 위해 알아야하는 개념에 대해 정리해보자.

![](https://velog.velcdn.com/images/willy4202/post/9215dd6e-e4fc-42d0-b07f-5b37f5a591f3/image.png)

## 서버란?

서버는 네트워크를 통해서 서비스를 하는 컴퓨터를 의미한다. 간단한 예로, 우리는 웹 브라우저를 통해 웹 사이트에 접근해 정보를 받을 수 있다. 이때, 웹 브라우저가 '클라이언트'이고, 웹 사이트의 콘텐츠를 제공하는 컴퓨터가 서버인 셈이다.

> “기술 스타트업들은 큰 뜻을 품은 기업가들이 자신의 차고에서 프로토타입을 만든 후에 마침내 큰 성공을 거둔다”

이전만 하더라도 서버를 관리하기 위해서는, 회사 내에 `서버실` 또는 `개발실`이라는 방이 따로 있었다고 한다. 구글, 애플과 같은 거대 it 기업들이 처음 시작은 차고에서 시작했다는 말이 있지 않나? 서버실을 관리하기 위해서는 신경써야할 것이`(온도, 습도 , 통풍 등...)` 한둘이 아니었다고 한다.

![](https://velog.velcdn.com/images/willy4202/post/64d5e810-5aac-4278-9e41-40ccb23d7842/image.png)

아무튼 서버 관리를 이어서 얘기해보자면, 서버는 특정 트래픽, 사람이 많이 몰리면 터졌다라는 말을 하기도 한다. 이때는 서버가 수용 할 수 있는 유저보다 더 많은 데이터가 몰리면서 과부하가 걸리는 상황을 말한다. 이를 방지하기 위해서 `scale-up`과 `scale-out`이라는 방법을 사용한다.

이 두 가지 방법을 전통적인 서버 관리법으로 설명해보자면, `scale-up`은 서버를 높은 사양으로 업그레이드 하는 것을 말한다. `scale-out`은 장비를 추가해서 확장하는 방식을 말한다. 쉽게 얘기해서 컴퓨터 사양을 올리거나, 컴퓨터를 하나 더 놓는 방식으로 서버를 증량할 수 있다.

이런 전통적인 방법을 보았을 때, 기존의 전통 서버 제공 방식의 한계점이 드러나기 시작했다. 바로 `온프레미스`방식 때문이다. 이 방식은 자체적으로 보유한 전산실 서버에 직접 설치해 서버를 운영하는 방식인데, 다음과 같은 문제점을 찾아볼 수 있었다.

1. datacenter의 렌트 비용을 지불.
   구독제로 사용해야했기 때문에, 회사가 망하든 성공하든 유동적으로 서버를 가져가기 어려웠다.

2. 전원, 쿨링, 유지보수 비용
   자신이 감당할 수 있는 것보다 많은 트래픽이 몰리게 되면, 과부하되기 쉬웠다.
3. 새로운 서버를 추가하거나 교체할 때 시간 소요

4. 인프라를 모니터링 할 인원이 항시 대기해야한다.

## 클라우드 컴퓨팅

이전의 서버 관리법에 문제가 많아지자, 클라우드 시스템이 새로운 대안으로 제시됐다.

`클라우드 컴퓨팅 : 인터넷을 통해 on-demand로 제공해주는 서비스`

1. 사용한만큼만 지불하면 되는 방식

2. 기존보다 간편하게 서버, 스토리지, DB에 접근가능한 솔루션을 제공한다.

3. 필요한 타입과 크기의 리소스를 지정하여 사용할 수 있다.

4. 고 가능성으로 글로벌하게 확장된 인프라

## AWS

그렇다면 클라우드 시장의 시작이자 1위를 유지하고 있는 AWS에 대해 알아보자. AWS는 Amazon Web Services 아마존의 클라우드 플랫폼을 뜻한다.

> 아마존 웹 서비스는 다른 웹 사이트나 클라이언트측 응용 프로그램에 대해 온라인 서비스를 제공하고 있다. 이러한 서비스의 상당수는 최종 사용자에 직접 공개되는 것이 아니고, 다른 개발자가 사용 가능한 기능을 제공하는 플랫폼을 제공하는 PaaS이다. (출처:링크)

![](https://velog.velcdn.com/images/willy4202/post/c8a2a0fe-0b78-4b98-a58f-23546003ceb2/image.png)

AWS를 사용하는 이유는 다음과 같다.

1. 클라우드 컴퓨팅의 장점을 살렸다.
2. 간단한 가입 프로세스
3. 브랜드 파워(?)
4. stabilty - 규모에 비해 안정적

### Key-services

1. EC2 (elastic copute cloud) - 몇 분안에 구동 가능한 가상서버

2. VPC (virtual private cloud) - AWS 네트워크 망안의 사용자 전용의 사설 네트워크 망

3. S3 (simple sotrage serivde) - 파일 업로드 및 공유
   거의 모든 회사들이 사용하는 기술이라 생각해도 될 정도로 핵심 키 서비스다.
   구글 드라이브와 비슷한 개념이다. 어떤 데이터든 저장이 가능하다.
   데이터적인 관점으로 보자면, 사진 URL 그대로 데이터에 저장할 수 있다.
   사진을 저장한 상태에서 상태에서 주고받으면 상당히 용량이나, 통신 속도 측면에서 비효율적이다.

4. CDN - cloudFront - 글로벌한 서비스의 경우 레이턴시가 생길 수 있다. 해당 기술을 사용하면, 미리 캐싱된 데이터를 전송하는 방식으로 속도를 현저히 증가시킬 수 있다.
   캐싱이 무엇인지에 대해서 따로 정리해두자.

5. DNS (도메인 네이밍 시스템)- Route 53 : 도메인은 건물 이름이고, 호스트는 그 건물에 사는 사람, ip는 건물의 주소라고 생각하면 된다. 이 기술을 활용하면 호스트의 도메인을 다른 이름으로 바꿀 수 있다.

6. RDS (relational database Service) : 클라우드에서 데이터 베이스 관리

7. ELB (Elastic load balanacing) : 서버로 들어오는 트래픽을 여러개의 머신으로 분산해주는 기술, 서버 차원에선 부담을 줄여줄 수 있다.
